小技巧_u3d_C#

Created2022-06-28|Updated2022-06-28|Unity
|Post View:

1，使用Physics.OverlapSphereNonAlloc，不会像Physics.OverlapSphere那样创建出Collider[]数组出来，它生成的是一个int值类型，这意味着就不会分配内存空间，不会产生GC，很多检测都有NonAlloc，有些情况可以考虑使用这些OverLapXXXNonAlloc。

2，对于两个脚本间互相调用，会遇到start与start间初始化时间存在乱序而空引用的错误，此时可以用访问器解决，

xxx xx get{
	if(_xx==null){
		_xx = GetComponent<xxx>();
	}
	return _xx;
}

3，yield return new 产生的GC问题。可以生成一个静态对象解决。

public static class YieldHelper
{
		public static WaitForEndOfFrame WaitForEndOfFrame = new WaitForEndOfFrame();
		
public static IEnumerator WaitForSeconds(float totalTime,bool ignoreTimeScale = false)
{
		float time = 0;
		while(time < totalTime)
{
		time += (ignoreTimeScale?Time.unscaledDeltaTime:Time.deltaTime);
	  yield return null;
}
}
}

4，enum : byte 的作用
这个表示枚举元素使用 byte 存储。
普通声明的话，是继承int。例 public enum Icon_Type : int
继承自byte可以稍微减少一点内存占用。

5，对比int比对比string效率更高，所以有些地方可以考虑用将字符串转化为哈希值再对比。当然可能有哈希冲突的风险。
