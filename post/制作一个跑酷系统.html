制作一个跑酷系统

Created2022-06-26|Updated2022-06-28|Unity
|Post View:

1，人物控制和相机控制

1：控制相机与角色的偏移

transform.position = new Vector3(x,x,x)+ followTarget.position;

2：让相机以角色为中心点，以一个圆为周期转动。


transform.position = Quaternion.Euler(x轴转向, y轴转向, 0) * new Vector3(0,0,距离角色距离) 
+ followTarget.position + 中心点的偏移量;

transform.rotation = Quaternion.Euler(x轴转向, y轴转向, 0)

3：移动和根据相机位置调整方位

知识点 : 四元数 * 向量 = 旋转后的向量 （注意左右乘）

//监控按键 0渐变为1或者-1
float h = Input.GetAxis("Horizontal");
float v = Input.GetAxis("Vertical");

//监控按键的量，有值说明有按键
float moveAmount = Mathf.Clamp01(Mathf.Abs(h) + Mathf.Abs(v));

//构建移动向量，注意椭圆映射
var moveInput = (new Vector3(
            h * Mathf.Sqrt(1 - Mathf.Pow(v,2) / 2.0f), 
            0, 
            v * Mathf.Sqrt(1 - Mathf.Pow(h, 2) / 2.0f)));

//获得相机的旋转量 并乘上移动向量，获得旋转后向量
var moveDir = cameraController.PlanarRotation * moveInput;

//若有按键
if (moveAmount > 0) 
        {   //移动
            transform.position += moveDir * moveSpeed * Time.deltaTime;
            //保存转动方向
			targetRotation = Quaternion.LookRotation(moveDir);
         
        }

//转动角色朝向
transform.rotation= Quaternion.RotateTowards(
transform.rotation, targetRotation, rotationSpeed * Time.deltaTime);
	
//设置混合树 
//dampTime: 用 dampTime 长的时间，将 我们设置的叫做  name 的 float类型 的值由 原本的值 改变到给定的 value 值。
//deltaTime：两次执行该方法的时间间隔。（因为这个方法会每 deltaTime 执行一次，直到 name = value）
animator.SetFloat(Animator.StringToHash("NormalAmount"), moveAmount,0.2f,Time.deltaTime);

4：转用CharacterController来控制角色移动

CC自带有胶囊碰撞体。一般来说这类东西的组合是Rb+Collider或者单独一个CC。CC适用于第一人称或者第三人称人物角色控制，但是需要自己写一些东西比如重力。

CharacterController 

1.Character Controller不是基于物理，没有重力，没有阻力，没有加速度，所有移动都是精确的

2.Character Controller可以跨过物体，也就是上台阶，不过会一顿一顿，同是下楼梯需要自己写重力

3.Character Controller 不会对 Rigidbody产生反应

4.Character Controller自带地面监测，不过好像有BUG，一般还是自己写

5.Character Controller的移动通过向量控制，自带碰撞监测

Rigidbody

1.Rigidbody是基于物理的，所以移动会比较平滑自然，但是因为有各种力，所以很难保证精确度

2.Rigidbody会对Rigidbody产生反应

3.Rigidbody的移动通过力或者速度控制，不加碰撞体就不会产生碰撞


//transform.position += moveDir * moveSpeed * Time.deltaTime;
//改成
characterController.Move(moveDir * moveSpeed * Time.deltaTime);

5：使用Physics物理检测

检测是否踩在地板上，并且添加重力影响。

[SerializeField] float groundCheckRadius;
[SerializeField] Vector3 groundCheckOffset;
[SerializeField] LayerMask groundLayer;
[SerializeField] bool isGround;
GroundCheck(); 
//如果踩在地板上
if (isGround) 
{
   ySpeed = -0.5f;
}else{
	//添加一个向下的量，用以模拟重力掉落。
   ySpeed += Physics.gravity.y ;
}
var velocity = moveDir * moveSpeed;

//添加向下的量
velocity.y = ySpeed;

//
characterController.Move(velocity * Time.deltaTime);

private void GroundCheck() {
//射线检测，创建一个球体用来检测
    isGround = Physics.CheckSphere(transform.TransformPoint(groundCheckOffset) ,
    groundCheckRadius, 
    groundLayer.value);
}

2.跑酷系统

1.墙面扫描

设定一个墙面扫描的脚本，用处是扫描人物面前的障碍物。

对于不同高度的障碍物，需要使用射线扫描面前的障碍物高度，然后根据高度选择播放的动画。

//前向射线的偏移值
[SerializeField] Vector3 forwardRayOffset;
//前向射线长度
[SerializeField] float forwardRayLength = 0.8f;
//扫描高度值的射线最大值
[SerializeField] float heightRayLength = 5f;
//用于标识障碍物的层
[SerializeField] LayerMask obstacleLayer;

public ObstacleHitData ObstacleCheck() 
{
    var hitData = new ObstacleHitData();
    var forwardOrigin = transform.position + forwardRayOffset;
        
    //前向射线
    hitData.forwardHitFound = Physics.Raycast(forwardOrigin, 
    transform.forward,
    out hitData.forwardHit, forwardRayLength,obstacleLayer);

    Debug.DrawRay(forwardOrigin, transform.forward * forwardRayLength,(hitData.forwardHitFound) ?Color.red:Color.white);

    if (hitData.forwardHitFound) {
        //碰撞点上面往下射,刚好碰到面前物体的边缘,算出障碍物的总高度,需要减去人物最低点才能得到实际高度。
        var heightOrigin = hitData.forwardHit.point + Vector3.up 
        * heightRayLength;

        hitData.heightHitFound = Physics.Raycast(heightOrigin,Vector3.down,
        out hitData.heightHit,heightRayLength,obstacleLayer);

        Debug.DrawRay(heightOrigin,Vector3.down * heightRayLength,(hitData.forwardHitFound) ? Color.red : Color.white);

        Debug.Log(hitData.heightHit.point);
    }
        return hitData;
}

public struct ObstacleHitData {
    //前向射线是否有碰撞
    public bool forwardHitFound;
    //高度射线是否有碰撞
    public bool heightHitFound;
    //前向射线
    public RaycastHit forwardHit;
    //高度射线
    public RaycastHit heightHit;
}

2.播放动画

首先封装一个行为类ParkourAction，继承自ScriptObject，用于存储动画信息和触发动画需要的条件。

[SerializeField] private  string animName;
//可播放动画的最低高度
[SerializeField] private float minHeight;
//可播放动画的最高高度
[SerializeField] private float maxHeight;

//是否要角色转向障碍物
[SerializeField] bool rotateToObstacle;

[Header("使用Target Matching")]
[SerializeField] bool enabledTargetMatching = true;
[SerializeField] AvatarTarget matchBodyPart;
[SerializeField] float matchStartTime;
[SerializeField] float matchTargetTime;

//存储四元数,里面是射线碰撞到的障碍物的法线方向取反,即角色朝向障碍物面的四元数。
public Quaternion TargetRotation { get; set; }
public Vector3 MatchPos { get; set; }

//检测条件是否符合播放动画的要求
public bool CheckIfPossible(ObstacleHitData hitData, Transform player) {
    //减去人物的高度得出障碍物的高度
    float height = hitData.heightHit.point.y - player.position. y;
    Debug.Log($"面前的障碍物高度为:{height}");
    
    if (height <= minHeight || height > maxHeight) {
        return false; 
    }

    if (rotateToObstacle)
    {
        
        TargetRotation = Quaternion.LookRotation(-hitData.forwardHit.normal);
    }
       
    if (enabledTargetMatching) {
        MatchPos = hitData.heightHit.point;
    }
    return true;
}

public string AnimName => animName;
public bool RotateToObstacle => rotateToObstacle;

public bool EnabledTargetMatching => enabledTargetMatching;
public AvatarTarget MatchBodyPart=> matchBodyPart;
public float MatchStartTime => matchStartTime;
public float MatchTargetTime => matchTargetTime;

对于跨越或者攀爬这种会实际产生模型位移的动画，需要注意Animator的Apply Root Motion和动画Import Setting里Animation的Bake Into Pose，此处需要将Animator组件的Apply Root Motion勾选上。
使用协程来执行跨越障碍物的动画，

IEnumerator DoParkoutAction(ParkourAction action) {
    //是否在播放动画
    inAction = true;
    //角色控制器是否可控
    playerController.SetControl(false);
    animator.CrossFade(action.AnimName, 0.1f);
    //animator.Play("StepUp");
    yield return null; 

    
    //此函数在过渡期可用,获得下一个动画状态
    var animState = animator.GetNextAnimatorStateInfo(0);
    if (!animState.IsName(action.AnimName)) {
        Debug.Log("The parkour animation is wrong");
    }
         

    float timer = 0f;
    //动画播放中
    while(timer <= animState.length)
    {
        timer += Time.deltaTime;

        if (action.RotateToObstacle) {
            //转向障碍物
            transform.rotation = Quaternion.RotateTowards(
            transform.rotation,
            action.TargetRotation,
            playerController.RotationSpeed * Time.deltaTime);
        }

        //目标匹配,后面再讲
        if (action.EnabledTargetMatching) {
            MatchTarget(action);
        }

        yield return null;
    }

    //解除锁定
    inAction = false;
    playerController.SetControl(true);
    }

3.优化动画

出现的问题：

  1. 动画在播放结束后，脚部会出现障碍物穿模或者高过障碍物的情况。

  2. 有攀墙动作的动画，手部无法真正抓住边缘。

  3. 攀爬结束后按前进会导致人物平移一段时间。

  4. 攀爬过程中整个人穿模。

Target Matching 目标匹配

Unity手册的描述：通常在游戏中可能出现以下情况：角色必须以某种方式移动，使得手或脚在某个时间落在某个地方。例如，角色可能需要跳过踏脚石或跳跃并抓住顶梁。这种情况可以用Target Maching。(似乎只能用在AnimatorController里)

我们的动画里有跃起，然后踏上障碍体的动作。首先要在AnimationClip里找到动画标准化时间的两个点，1个是角色离地的时间点，1个是踏上障碍体,结束落地的时间点。

void MatchTarget(ParkourAction action) {
    //是否要使用了目标匹配
    if (animator.isMatchingTarget) {
        return;
    }

    //参数解释：1.需要匹配的位置(一般为障碍物的高度值y,这里为高度射线的碰撞点位置取y值)
    //2.角色转向的四元数。 3.系统枚举值:要匹配的肢体,人物四肢的其中1个。 4.~。 5.开始匹配的时间点。 6.结束匹配的时间点
    animator.MatchTarget(action.MatchPos,transform.rotation,
        action.MatchBodyPart,
        new MatchTargetWeightMask(new Vector3(0,1,1),0),
        action.MatchStartTime,
        action.MatchTargetTime);
    }

第1点，这时候可以调整,调为feet。

使用 Feet__ 时，对于所有帧，Root Transform Position Y 将与位置最低的脚 Y 匹配。因此，混合点始终保持在脚周围，从而防止在混合或过渡时发生浮动问题。

脚超模的现象可以按第2点调。

第2点，需要调整MatchTarget第4个参数，Unity手册里写的是：包含匹配位置和旋转的权重的结构，使用此结构可为 Animator.MatchTarget 指定位置和旋转权重遮罩。

有一个Vector3类型的参数positionXYZWeight:位置 XYZ 权重,值在0~1之间。

一个float类型的参数rotationWeight: 旋转权重

这里把positionXYZWeight的z值设为最高值1,这样匹配的权重会偏向障碍物平面的最边缘，可以把这个权重值封装在ScriptObject行为类里。
同理把y值设为1,说明在匹配y值方面,会把权重偏向最接近障碍物平面上,这样就不容易出现脚穿模在地下的情况。

第3点，这里有一点上述的代码里没讲，因为攀爬动作没有从蹲下到站立的动画，所以攀爬结束后混合了一段蹲到站立的动画。而这里是由于使用了Time.deltaTime只等待了一段AnimationClip的播放，没有等待蹲下到站立的部分。这段的解决比较宽泛，可以用yield return 再等待一段时间。

第4点，这个是因为Match Target的匹配时间点没设好，只能慢慢调了。
